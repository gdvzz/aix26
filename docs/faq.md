# 一些建议 Q&A

`更新-260326` \| `发布-260326`

记录参加大赛同学提出的部分问题（Question），以及对应的建议（Advice），供同学参考。如需协助，欢迎联系董老师。发消息/面聊（大多在B102），都可以。

有几个来源：

- 大赛启动会

## ℹ️ **当前需要学习什么？**

✅ 建议如下：

- 开发板。作为一台新电脑，要熟悉下，至少会开关机。
- 操作系统。几种开发板都安装了 Linux Ubuntu，和 Windows 不同。至少要会常用的操作。
- Python。也有要所了解，因为人工智能 App 会用到 Python。
- 其他和选定课题相关的知识。
- ✳️ 相关知识/技能，支撑完成大赛就好。不一定在大赛期间成为某个知识/技能的专家。
- 如需协助，可联系董老师。

## ℹ️ **应该怎么学？**

✅ 建议如下：

- 以终为始。缺什么，就学什么。
- 可以让 AI 写些代码，但要能看懂，知道这些代码的用途。
- 看不懂时，让 AI 注释/解释代码。然后按需翻翻某语言的教程。
- 要能分清楚：哪些是自己要搞定的，哪些是可以借力的。
- 多交流、多沟通、多求助。
- 如需协助，可联系董老师。

## ℹ️ **有了个不错的构想。怎么着手开始？**

✅ 建议如下：

- 从小处开始。比如构想要处理 5000 种手势，则先尝试处理识别比如“翘大拇指的夸赞”手势。
- 不一定要制造作品的所有部件。可以找找有没有开源的项目，可以作为最终作品的组成部分。
- 不一定每个功能，都要用到 AI。比如就只能处理 10 种手势，手势1就是你好，手势2就是再见。
- 先尝试做个很不完善的最小、最简的原型。然后不断优化、迭代。
- 先关注作品的完整性。不要太纠结AI某指标的数值。
- 分工 + 协作。
- 千里之行，始于足下。
- 大想法为指导，由于时间、人力有限，最终可先做个小作品。
- 小想法为起点，在做的过程中得到更多信息，最终可能做出大作品。
- 如需协助，可联系董老师。

## ℹ️ **某某技术，能不能实现某某功能？**

✅ 建议如下：

- 找相关资料做参考，得出基本判断。
- 如有相关开源项目，通过运行开源项目就可以得到实证。
- 不一定一步到位采用目标技术和方法（可能太复杂）。可先尝试低精度、高成熟的“替代”方案。
- 如需协助，可联系董老师。
